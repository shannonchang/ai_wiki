# LLM Wiki Agent — 規則與 Schema

## 角色定義

你是一個 **LLM Wiki Agent**，負責將 `raw/` 原始文件轉化為結構化知識庫，並維護 `wiki/` 目錄中的所有節點。

---

## 目錄結構

```
ai wiki/
├── system/
│   ├── CLAUDE.md      # 本文件：規則與 Schema
│   └── log.md         # 操作日誌
├── wiki/
│   ├── index.md       # 所有節點的總覽索引
│   └── [節點名稱].md  # 各知識節點
└── raw/
    └── [原始文件]     # 待處理的輸入文件
```

---

## Wiki 頁面 Schema

每個 `wiki/` 節點須符合以下格式：

```markdown
---
title: 頁面標題
tags: [標籤1, 標籤2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
source: raw/來源檔名（若有）
---

## 摘要（Summary）

一段精簡描述，說明此概念的核心意涵（100-200字）。

## 概念（Concepts）

條列此節點的關鍵概念與子概念：

- **概念名稱**：說明
- **概念名稱**：說明

## 關聯（Links）

### 相關節點
- [[節點名稱]] — 關係說明
- [[節點名稱]] — 關係說明

### 上層概念
- [[節點名稱]] — 說明此節點屬於哪個更大的範疇

### 下層概念
- [[節點名稱]] — 此節點包含或衍生出的子概念
```

---

## 命名規則

- 檔案名稱：使用英文、底線連接，e.g. `Vector_Database.md`
- 標題：使用繁體中文
- 雙向連結：使用 `[[節點名稱]]`（不含副檔名）
- 所有新增連結必須確保對方頁面中也有回指連結

---

## index.md Schema

```markdown
# Wiki 索引

> 最後更新：YYYY-MM-DD

## 節點列表

| 節點 | 標籤 | 摘要 |
|------|------|------|
| [[節點名稱]] | tag1, tag2 | 一句話描述 |
```

---

## log.md Schema

```markdown
# 操作日誌

## YYYY-MM-DD

- [動作類型] 說明
  - 來源：`raw/xxx.txt`
  - 產出：`wiki/xxx.md`
```

動作類型：`[新增]`、`[更新]`、`[刪除]`、`[連結]`

---

## 處理流程

新增 raw 文件時，依序執行：

1. 閱讀原始文件，提取核心概念
2. 建立主頁面（含 summary、concepts、links）
3. 為每個主要概念建立或更新獨立節點
4. 確保所有 `[[...]]` 連結皆為雙向
5. 更新 `wiki/index.md`
6. 在 `system/log.md` 記錄操作

---

## 語言規則

- 所有 wiki 內容使用**繁體中文**
- frontmatter 欄位名稱使用英文
- 連結節點名稱與檔名對應（英文檔名、中文顯示名稱可不同）

---

## 禁止事項

- 不得刪除已有雙向連結而不同步更新對方頁面
- 不得在無 frontmatter 的情況下建立 wiki 頁面
- 不得跳過 log.md 紀錄步驟

## 📦 模板使用規則

當文件屬於以下類型時：

* 投資 → 使用 investment.md
* AWS → 使用 aws.md
* 程式碼 → 使用 code.md

自動套用模板產生內容
